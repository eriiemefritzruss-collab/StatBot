# 人工智能导论 期末项目考核要求

## 一、项目概述

本课程期末项目要求各小组设计并实现一个具有实际应用价值的 Agent 系统。项目方向应与以下领域相关：

- **AI4S（AI for Science）**：利用人工智能方法解决科学研究中的问题，如分子设计、材料发现、科学文献分析、实验数据分析等
- **AI + 统计**：结合统计方法与AI技术，如智能统计分析、自动化统计推断、统计模型选择等
- **机器学习相关**：自动化机器学习流程、智能特征工程、模型调参Agent、数据分析Agent等

### 项目要求

项目需体现 Agent 的核心特性，包括但不限于：
- 自主决策与规划能力
- 工具调用与环境交互能力
- 记忆与上下文管理能力
- 多步骤任务执行能力

## 二、提交形式与要求

### 2.1 提交内容

各小组需提交以下内容：

**线上提交：**
- 技术报告（PDF 格式）
- PPT
- Poster
- PPT演示视频（不超过10分钟）
- 代码：压缩包或线上代码仓库链接（GitHub/GitLab）

**线下提交：**
- 技术报告（纸质版）


### 2.2 代码仓库要求

代码仓库应包含以下内容：

```
project-root/
├── README.md           # 项目说明：背景、安装、运行方法
├── requirements.txt    # Python依赖（或其他语言的依赖配置）
├── src/                # 源代码目录
├── config/             # 配置文件（如有）
├── data/               # 示例数据或数据获取脚本（如有）
├── tests/              # 测试代码（如有）
└── docs/               # 额外文档（如有）
```

**README.md 必须包含：**
- 项目背景与目标
- 环境配置与安装步骤
- 运行方法与示例
- 小组成员信息

### 2.3 技术文档要求

技术文档应包含以下章节（建议 8-15 页）：

1. **引言**
   - 项目背景与动机
   - 问题定义与目标

2. **相关工作**
   - 相关领域现有方法概述
   - 本项目的创新点或改进之处

3. **方法设计**
   - Agent 架构设计（建议附架构图）
   - 核心算法与技术选择
   - 工具与环境设计

4. **实现细节**
   - 关键模块实现说明
   - 遇到的技术难点及解决方案

5. **实验与结果**
   - 实验设置与数据
   - 评估指标与方法
   - 结果分析与讨论
   - 案例展示（Agent运行示例）

6. **总结与展望**
   - 项目总结
   - 局限性分析
   - 未来改进方向

7. **参考文献**

8. **附录**（可选）
   - 详细实验数据
   - 代码片段说明
